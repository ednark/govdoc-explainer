FAM 103.4 DATA USE




    Raw Data

    Source Text



    Summary of Government Standards Document

    Primary Standards, Guidelines, and Requirements: 1. Purpose and Compliance: - Data must be used according to the signed MOU or DSA. - Any other use must be justified, documented, and approved.

    1. Authoritative Data:

      • Use authoritative data when available.

    2. Points of Contact:

      • Know who to contact for permissions and restrictions related to data use.

    3. Regular Audits:

      • Conduct regular audits to ensure data use complies with its intended purpose.

    4. Data Use Logging:

      • Log data use to make it auditable.

    5. Clearance Inclusion:

      • Include the relevant Data Stewards from other offices when using their data.

    Key Focus Areas:

    Security: - Ensure data use is documented and approved if it deviates from the official purpose. - Log data use for audit purposes.

    Compliance: - Use data according to the signed agreements (MOUs or DSAs). - Conduct regular audits to ensure compliance. - Include appropriate Data Stewards in the clearance process when using external data.

    Accessibility: - Not explicitly mentioned in the standards.

    User Experience: - Ensure clear communication about whom to contact for data permissions and restrictions.



    Purpose: The document ensures proper and compliant use of data by monitoring, documenting, and approving its use.

    Punchline Summaries:

    • DevOps Technician: "Make sure all data use infrastructure logs and audits data access to stay compliant."
    • Designer: "Ensure user flows and interfaces comply with documented and approved data use guidelines."
    • Developer: "Incorporate checks to confirm data used follows set rules and is logged for audit."
    • Project Manager: "Track and ensure all team data use adheres to documented purposes and gains proper approvals."


    For a DevOps technician, focusing on infrastructure to support compliance, the following actions must be taken to align with the government standards:

    1. MOU/DSA Compliance: Ensure implementation details align with the Memorandum of Understanding (MOU) or Data Sharing Agreement (DSA) for data use.
    2. Authoritative Data Management: Use and highlight authoritative data sources within the infrastructure settings.
    3. Data Use Communication: Implement clear logging and communication channels detailing what data use is permitted.
    4. Regular Audits: Set up automated scripts for regular audits of data access and use.
    5. Audit Logging: Enable and maintain comprehensive logging for all data access and manipulation activities.
    6. Cross-Bureau Collaboration: Add mechanisms to notify and involve Data Stewards from other bureaus/posts when their data is used in the application.


    Here's a minimal list of actions a Designer should take on a project to show compliance with the government data use standards:

    1. Clarify Data Purpose: Ensure the purpose of data use is clearly communicated in the UI, referencing the MOU or DSA where necessary.

    2. Source Indication: Display the source of authoritative data within the user interface, ensuring users know it is reliable.

    3. Contact Information: Integrate straightforward, easily accessible contact information for users to get guidance on data usage.

    4. Compliance Notices: Include notices or reminders within the UI regarding the appropriate and approved use of data to promote regular awareness.

    5. Audit Prompts: Design features that facilitate regular checks and auditing of data usage by users.

    By following these steps, designers can ensure their work contributes to the project's compliance with data use standards.



    To ensure compliance with the standards from a developer's perspective, particularly one specializing in implementing backend business logic in Drupal, the following minimal actions should be taken:

    1. Use Authorized Data Only:

      • Integrate data sources specified in the MOU or DSA.
      • Verify and document the data source is authoritative.

    2. Purpose-Specific Data Usage:

      • Implement logic ensuring data is used solely as specified in the statement of purpose.
      • Prevent repurposing data without necessary approvals.

    3. Access Guidance for Data Use:

      • Provide clear in-application guidance or contact information regarding data use restrictions.

    4. Audit and Logging Mechanisms:

      • Install and configure modules for logging data usage.
      • Regularly schedule and perform audits to ensure usage complies with the agreed purposes.

    5. Include Relevant Data Stewards:

      • Ensure applications add notifications to appropriate Data Stewards when their data is used.

    By following these actions, you can ensure your application development aligns with the provided government standards.
