    [0] Home Guides and Resources Connected Government Act Connected Government Act As of July of 2018, all new and redesigned agency websites are required to be mobile-friendly. The Connected Government Act (H.R.2331) was signed into law on January 10, 2018, and requires new and redesigned federal agency public websites to be mobile-friendly. The General Services Administration (GSA) and Office of Management and Budget (OMB) submitted a report to Congress in 2019 that describes how agencies can implement the law and assess their compliance. Law H.R.2331 : Connected Government Act To require a new or updated Federal website that is intended for use by the public to be mobile friendly, and for other purposes. (NOTE: Jan. 10, 2018 - [H.R. 2331]) Be it enacted by the Senate and House of Representatives of the United States of America in Congress assembled, (NOTE: Connected Government Act. 44 USC 101 note.) SECTION 1. SHORT TITLE. This Act may be cited as the “Connected Government Act”.
    [1] SEC. 2. FEDERAL WEBSITES REQUIRED TO BE MOBILE FRIENDLY. (a) Amendment. –Subchapter II of chapter 35 of title 44, United States Code, is amended by adding at the end the following new section: “Sec. 3559. (NOTE: 44 USC 3559.) Federal websites required to be mobile friendly “(a) (NOTE: Time period.) In General.– If, on or after the date that is 180 days after the date of the enactment of this section, an agency creates a website that is intended for use by the public or conducts a redesign of an existing legacy website that is intended for use by the public, the agency shall ensure to the greatest extent practicable that the website is mobile friendly. “(b) Definitions.–In this section: “(1) Agency.–The term ‘agency’ has the meaning given that term in section 551 of title 5.
    [2] “ (2) Mobile friendly.–The term ‘mobile friendly’ means, with respect to a website, that the website is configured in such a way that the website may be navigated, viewed, and accessed on a smartphone, tablet computer, or similar mobile device. ”.     Summary of Key Standards, Guidelines, and Requirements

    Primary Standards and Requirements:

    1. Connected Government Act (H.R.2331):

      • Requires new/revised federal websites to be mobile-friendly.
      • Mobile-friendly means usable on smartphones, tablets, and similar devices.

    2. Mobile Development Strategies:

      • Mobile-first, mobile-only, mobile-friendly strategies.
      • Develop mobile products and services for anytime, anywhere access.

    3. Google's Mobile-Friendly Ranking:

      • Google boosts rankings for mobile-friendly sites.

    4. Responsive Web Design:

      • Ensures web pages scale from mobile to desktop.

    5. Structured Content and APIs:

      • Core for open content platforms and data sharing.

    Security:

    1. Making Your Agency Apps Secure:

      • Guidelines for app security and user data protection.
      • DHS report on mobile device security threats and recommendations.

    2. CIO Council Federal Secure Mobility Toolkit:

      • Best practices report for securing mobile initiatives in federal agencies.

    3. Privacy Guidelines:

      • Privacy risk assessments throughout development and information lifecycle.

    Accessibility:

    1. Federalist Hosting Solution:

      • Provides templates compliant with Section 508 Standards.

    2. Accessibility Practices:

      • Self-testing and accessibility guidelines for designers, developers, and product managers.

    User Experience:

    1. Mobile User Experience Guidelines:

      • Six key guidelines to ensure good mobile user experience.

    2. Mobile First Webinar:

      • Importance of designing for mobile devices first.

    3. Push Notifications:

      • Effective in real-time communication with users.

    4. Mobile Content Recommendations:

      • Less is more for mobile content.

    5. Mobile Paper Prototype:

      • Early-stage design and testing with mockups.

    Compliance:

    1. GAO Report and U.S. Digital Services Playbook:

      • Guidelines and strategies for maintaining compliance with digital standards.

    2. Digital Analytics Dashboard:

      • Tool for monitoring and improving website performance and compliance.

    3. Federal Crowdsource Mobile Testing Program:

      • Free service for compatibility testing on various devices and operating systems.

    By focusing on simplicity, mobile adaptability, security measures, accessibility, and user experience, these standards ensure federal web applications are user-friendly and compliant with current regulations.

    To ensure compliance with the Connected Government Act and related federal standards from a DevOps technician's perspective, here are the essential actions to take:

    1. Ensure Mobile-Friendliness:

      • Confirm that the web application adapts to various screen sizes (smartphones, tablets, desktops).

    2. Implement Responsive Design:

      • Use flexible grids, flexible images, and CSS3 media queries to ensure the site works well on mobile devices.

    3. Performance Optimization:

      • Optimize load times for mobile devices using tools like performance testing and predictive rendering.

    4. Set Up Mobile-Friendly Testing:

      • Utilize mobile emulators and real device testing to confirm the site performs well on actual mobile hardware.

    5. Accessibility Compliance:

      • Build and test following Section 508 standards to ensure accessibility for users with disabilities.

    6. Security Measures:

      • Implement strong security practices for mobile access, including HTTPS, data encryption, and robust authentication.

    7. Analytics Integration:

      • Embed analytics tools to monitor mobile usage and performance metrics, ensuring consistent user experience across devices.

    8. Continuous Deployment Pipeline:

      • Set up a CI/CD pipeline that includes automated testing for mobile responsiveness and accessibility during the deployment process.

    9. Server-Side Adaptations:

      • Configure servers to deliver optimized content based on device detection (e.g., different image sizes and resources for mobile users).

    10. Scalability:

      • Ensure the infrastructure can scale to handle increased mobile traffic without performance degradation.

    By following these steps, a DevOps technician can help ensure the web application complies with the Connected Government Act requirements and related federal standards for mobile access.



    Minimal List of Actions for Compliance as a Designer:

    1. Adopt Mobile-First Design:

      • Ensure new or redesigned websites are built for mobile devices first.

    2. Use Responsive Web Design:

      • Implement fluid grid layouts, flexible images, and CSS3 media queries.

    3. Test for Mobile-Friendliness:

      • Verify navigation, view, and accessibility on smartphones, tablets, and similar devices using tools like Chrome DevTools.

    4. Incorporate Accessibility Standards:

      • Follow Section 508 Standards to ensure accessibility for all users.

    5. Simplify Content for Mobile:

      • Minimize and prioritize content to fit smaller screens effectively.

    6. Conduct Usability Testing:

      • Perform user testing specifically on mobile devices to identify and fix usability issues.

    7. Improve Load Speed:

      • Optimize images, minimize HTTP requests, and leverage browser caching to reduce load times.

    8. Collect and Analyze Mobile Analytics:

      • Use analytics tools to track mobile user behavior and adjust design accordingly.

    9. Follow Security and Privacy Best Practices:

      • Ensure the design adheres to guidelines for protecting user data and securing the application.

    10. Keep UI Consistent Across Devices:

      • Maintain a consistent design language to provide a seamless experience on different devices.
