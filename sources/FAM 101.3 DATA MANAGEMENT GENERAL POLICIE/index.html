    Summary of Government Standards Document

    Primary Standards, Guidelines, and Requirements:

    1. General Data Management Policies:

      • The Chief Data Officer (CDO) manages data policies with the Center for Analytics (M/SS/CfA).

    2. Openness of Data:

      • Data is presumed open unless restricted by privacy, security, confidentiality, or legal limitations.

    3. Enterprise Standards and Interoperability:

      • M/SS/CfA maintains data standards and ensures data can be shared across systems.

    4. Data Publishing:

      • Bureaus/offices must publish metadata about their data in the Enterprise Data Inventory, even if access is restricted.

    5. Data Archiving and Disposal:

      • Data must be archived or disposed of according to records retention policies.

    Highlighting Focus on Key Areas:

    1. Security:

      • Data must be restricted based on privacy laws, confidentiality, security, or legal agreements.

    2. Accessibility:

      • Data should be made available unless there are valid restrictions; metadata must be published even if data access is restricted.

    3. User Experience:

      • Guidance on preparing data for interoperability and consolidation across different systems to enhance usability.

    4. Compliance:

      • Data management must follow privacy laws (e.g., Privacy Act, FOIA), classification rules, and records retention schedules.


    Overall Purpose: Ensure government data is managed and shared effectively while maintaining security and compliance with legal requirements.

    Punchline Summaries:

    • DevOps Technician: You need to design and manage infrastructure to ensure data is secure, shareable, and compliant with legal standards.

    • Designer (Human Centered Design): Design the user experience to promote transparency and easy access to public data while ensuring compliance with privacy and security rules.

    • Developer (Drupal): Implement backend structures that facilitate data sharing and adhere to legal privacy and retention guidelines.

    • Project Manager: Coordinate tasks to ensure data management practices align with government policies and security requirements, ensuring project compliance and approval.



    Minimal Actions for Compliance:

    1. Data Management Policies:

      • Ensure alignment with CDO and M/SS/CfA directives.

    2. Openness and Privacy:

      • Implement controls to flag and restrict sensitive data as per privacy, confidentiality, and security guidelines.

    3. Data Availability:

      • Enable technical features to share non-exempt data internally.
      • Set up access controls and compliance checks for exempted data.

    4. Standards and Interoperability:

      • Use Master Reference Data (MRD) and comply with enterprise standards.
      • Facilitate data movement and consolidation with interoperability support.

    5. Metadata and Inventory:

      • Configure systems to automatically publish metadata to the Department’s Enterprise Data Inventory.

    6. Data Archiving and Disposal:

      • Implement mechanisms for data archiving and disposal as per records retention schedules (refer to 5 FAM 400).

    By completing these actions, a DevOps technician ensures the infrastructure supporting the web application complies with the relevant government standards.



    Sure, here are the minimal list of actions for a Designer to ensure compliance with the government standards for a web application:

    1. Adopt Open Design Principles:    - Favor openness in design where possible, balancing usability and security.

    2. Privacy by Design:    - Ensure PII and other sensitive information are handled according to privacy restrictions (e.g., mask or encrypt data fields).

    3. Accessibility:    - Follow accessibility standards (e.g., WCAG) to make the application usable for people with disabilities.

    4. User Metadata:    - Design interfaces that allow display and management of data metadata.

    5. Data Transparency:    - Incorporate features that clearly communicate any data sharing and privacy policies to users.

    6. Records Information:    - Design archiving and disposal options in accordance with records retention policies (e.g., clear user options and notifications for data archiving).

    7. Interoperability:    - Ensure design allows for data interoperability, such as modular interfaces for data import/export.

    By concentrating on these key actions, a designer can ensure that user interfaces and experience components of a web application are aligned with the relevant portions of the government standards.



    To ensure compliance with the relevant government standards from the perspective of a Drupal developer, here is a minimal list of actions:

    1. Data Openness Presumption: Ensure the web application is designed to prioritize data sharing and availability, while also respecting confidentiality, privacy, and security requirements.

    2. Data Privacy Protections: Implement measures to protect personally identifiable information (PII) according to the Privacy Act of 1974.

    3. FOIA Compliance: Make non-exempt data available for public access in compliance with the Freedom of Information Act (FOIA).

    4. Access Controls: Develop robust access control mechanisms to enforce data protection for sensitive information restricted by law, confidentiality agreements, and security requirements.

    5. Metadata Publication: Ensure that your web application includes functionality to publish metadata about the data being stored and managed, even if the actual data access is restricted.

    6. Data Preparation and Interoperability: Follow enterprise standards and guidance for preparing data for storage and use, enabling data movement and consolidation between systems.

    7. Records Management: Implement features that ensure data is either archived or disposed of according to the applicable records retention policies described in 5 FAM 400.

    8. Documentation: Include documentation within the application that justifies the non-sharing of certain datasets in the Department’s Enterprise Data Inventory, when applicable.

    By following these steps, a Drupal developer can ensure their web application complies with the relevant government data management standards.



    Sure! Here is a minimal list of actions a Project Manager should take to ensure compliance with the specified Government Standards for a web application:

    1. Appoint a Data Steward: Assign someone responsible for data management within the project.
    2. Data Identification and Classification: Identify and classify all data types being used, particularly noting any personally identifiable information (PII) or sensitive data.
    3. Data Openness Assessment: Ensure data not restricted by law, confidentiality, security, or other valid reasons is available for internal use.
    4. Metadata Documentation: Document metadata and include it in the Department’s Enterprise Data Inventory.
    5. Justification for Restricted Data: Provide justifications in the Enterprise Data Catalog for any data that cannot be shared.
    6. Adhere to Records Retention Policies: Ensure data is archived or disposed of in accordance with 5 FAM 400 records retention policies.
    7. Implement Interoperability Standards: Follow M/SS/CfA guidelines for data preparation and ensure compatibility for data movement and consolidation.
    8. Utilize Master Reference Data (MRD): Ensure alignment with enterprise standards via MRD and coordinate with Data Stewards and system developers.
    9. Regular Data Audits: Schedule and conduct regular audits to ensure ongoing compliance with data management policies.
