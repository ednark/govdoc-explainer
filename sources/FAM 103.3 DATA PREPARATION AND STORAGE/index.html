FAM 103.3 DATA PREPARATION AND STORAGE




    Raw Data

    Source Text



    Summary of Primary Standards and Requirements from Government Standards Document:

    1. Data Preparation and Storage    - Data intake must align with Master Reference Data (MRD) standards.    - Work with data creators to develop comprehensive metadata.    - Keep data as raw as possible.    - Identify and tag data as Classified or Sensitive But Unclassified (SBU).    - Store data and code transparently in a centralized location.    - Define storage location in the bureau's central repository.    - Maintain and publish metadata with data or code.    - Centralize and update data dictionaries regularly.    - Check data reliability with the Department’s Data Quality Program.

    2. Security    - Tag data as Classified or Sensitive But Unclassified (SBU).    - Follow classification guidelines for Classified data (5 FAM 480).    - Classified data must not be processed on OpenNet.    - Mark all Personally Identifiable Information (PII) as SBU.    - Encrypt data and include encryption keys if relevant.

    3. Accessibility    - Store data and metadata in a transparent and centralized manner.    - Define and publish the storage location in the central repository.    - Make data dictionaries accessible and update them at least every three years.

    4. Compliance    - Ensure data intake aligns with the Department's MRD standards.    - Include compliance-related metadata, such as Data Sharing Agreements.    - Include a statement on contractual limitations.    - Tag information that would be exempt from FOIA as SBU.

    5. User Experience    - Create metadata with a clear statement of purpose and contact information.    - Ensure all data or code is stored with appropriate metadata for clarity.

    To ensure compliance with the government standards specific to data preparation and storage, a DevOps technician managing infrastructure for a web application should:

    1. Align Data Intake:

      • Ensure data intake processes conform to the Department’s Master Reference Data (MRD) standards.

    2. Collaborate on Metadata:

      • Work with data users to ensure all necessary metadata (purpose, contact information, authoritative labels, data sharing agreements, encryption keys, contractual limitations) are developed and included.

    3. Maintain Raw Data Format:

      • Store data in its most raw and unprocessed form.

    4. Classify and Tag Data:

      • Mark and handle data according to its classification:
        • Classified data must be marked following 5 FAM 480 and must not be processed on OpenNet.
        • Sensitive But Unclassified (SBU) data, including PII, should be appropriately tagged and protected.

    5. Transparent and Centralized Storage:

      • Store data and code in a centralized and transparent manner:
        • Define and publish storage locations in the bureau’s repository.
        • Maintain and publish metadata with corresponding data/code.
        • Centralize and regularly update data dictionaries (every three years or when significant data changes occur).

    6. Verify Data Quality:

      • Use the Department’s Data Quality Program, if available, to ensure data reliability.

    By following these steps, a DevOps technician would be aligned with the essential data preparation and storage standards outlined in the government document.

    Certainly! Here are the minimal actionable steps a Drupal Developer must take to show compliance with the provided Government Standards Document:

    1. Align Data Intake with MRD Standards:

      • Ensure the data collected and stored aligns with the Department’s Master Reference Data standards (if available).

    2. Develop Metadata:

      • Collaborate with data users to develop and store metadata for datasets, including purpose, responsible contact information, authoritative label, data sharing agreements (if necessary), encryption keys, and contractual limitations.

    3. Store Data in Raw Format:

      • Keep data in its most original and raw format.

    4. Classify and Tag Data:

      • Tag datasets as Classified or Sensitive But Unclassified (SBU), including Personally Identifiable Information (PII).
      • Handle classified data according to 5 FAM 480 guidelines and ensure it is not processed on OpenNet.
      • Refer to 12 FAM 540 to identify typical SBU information.

    5. Centralize Data Storage:

      • Store data and accompanying code in a transparent and centralized location, ensuring both data and its metadata are published together to the bureau’s central repository.

    6. Maintain and Publish Data Dictionaries:

      • Centralize and publish existing data dictionaries.
      • Review/update data dictionaries every three years or whenever 5% or more of the data changes.

    7. Verify Data Reliability:

      • Consult the Department’s Data Quality Program, when available, to ensure data reliability.

    By following these specific actions, the developer can ensure the web application complies with the relevant parts of the Government Standards Document.
