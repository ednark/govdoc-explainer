## Document Summary: Secure Federal Websites through HTTPS

**Security:**

- Websites must enable HTTP Strict Transport Security (HSTS) to prevent insecure connections.
- All external resources must be loaded over HTTPS to ensure security.
- Agencies need to deploy HTTPS quickly to keep pace with evolving security vulnerabilities.

**Accessibility & User Experience:**

- Newly developed websites must use HTTPS from the start.
- Existing websites should prioritize HTTPS deployment based on risk, prioritizing sensitive data and high-traffic pages.
- All existing websites must be accessible through HTTPS by December 31, 2016.

**Compliance:**

- Agencies must deploy HTTPS on all domains and subdomains.
- Public dashboards are available to monitor compliance.
- Agencies should refer to resources at https://cio.gov for technical assistance.
